PHP源码分析-变量的存储方式

作者：迷茫者 发布于：2015-3-19 7:00 Thursday 分类：有道云笔记

PHP中的变量类型和值是通过C语言实现的，内核具体是如何组织用户在PHP中定义的变量呢？
HashTable在ZE内核中被广泛使用，PHP变量也正是存储在一个HashTable实现的符号表里。

当在PHP中调用一个函数或者类时，内核会创建一个新的符号表，这也是为什么在函数中无法使用函数外部定义的变量的原因。（因为他们分属两个符号表，一个当前作用域，一个全局作用域）


现在来看内核中是如何定义作用域的：

[cpp]  view plain copy
  1. struct _zend_executor_globals {  
  2.     //略  
  3.     HashTable symbol_table;//全局变量的符号表  
  4.     HashTable *active_symbol_table;//局部变量的符号表  
  5.     //略  
  6. };  

可以通过EG宏来访问变量符号表,EG(symbol_table)访问全局作用域的变量符号表，EG(active_symbol_table)访问当前作用域的变量符号表。


[php]  view plain copy
  1. <?php  
  2. $foo='bar';  
  3. ?>  

上面这段代码很简单，创建变量foo，并赋值bar。之后的PHP代码中就可以调用$foo变量了。
现在看看PHP中定义的变量，内核中是如何实现的。伪代码：
[cpp]  view plain copy
  1. zval* foo;  
  2. MAKE_STD_ZVAL(foo);  
  3. ZVAL_STRING(foo, 'bar', 1);  
  4. ZEND_SET_SYMBOL( EG(active_symbol_table), 'foo', foo);  

第一步、创建一个zval结构，并设置类型。
第二步、赋值为bar。
第三步、将其加入当前作用域符号表，只有这样用户才能在PHP里使用这个变量。

通过简单的这三步，即可实现定义PHP变量。简单的原因，在于内核为我们提供了强大的宏。现在我们将宏分别展开。

MAKE_STD_ZVAL应用到的宏有：

[cpp]  view plain copy
  1. #define     MAKE_STD_ZVAL(zv)               ALLOC_ZVAL(zv);INIT_PZVAL(zv)  
  2. #define     ALLOC_ZVAL(z)                   ZEND_FAST_ALLOC(z, zval, ZVAL_CACHE_LIST)  
  3. #define     ZEND_FAST_ALLOC(p, type, fc_type)       (p) = (type *) emalloc(sizeof(type))  
  4. #define     INIT_PZVAL(z)                       (z)->refcount__gc = 1;(z)->is_ref__gc = 0;  
MAKE_STD_ZVAL(foo)展开后得到：
[cpp]  view plain copy
  1. (foo) = (zval *) emalloc(sizeof(zval));  
  2. (foo)->refcount__gc = 1;  
  3. (foo)->is_ref__gc = 0;  
可以看出，MAKE_STD_ZVAL做了三件事：分配内存、初始化zval结构中的refcount、is_ref。

ZVAL_STRING应用到的宏有：

[cpp]  view plain copy
  1. #define ZVAL_STRING(z, s, duplicate) {  \  
  2.         const char *__s=(s);            \  
  3.         Z_STRLEN_P(z) = strlen(__s);    \  
  4.         Z_STRVAL_P(z) = (duplicate?estrndup(__s, Z_STRLEN_P(z)):(char*)__s);\  
  5.         Z_TYPE_P(z) = IS_STRING;        \  
  6.     }  
  7. #define Z_STRLEN_P(zval_p)      Z_STRLEN(*zval_p)  
  8. #define Z_STRLEN(zval)          (zval).value.str.len  
  9. #define Z_STRVAL_P(zval_p)      Z_STRVAL(*zval_p)  
  10. #define Z_STRVAL(zval)          (zval).value.str.val  
  11. #define Z_TYPE_P(zval_p)            Z_TYPE(*zval_p)  
  12. #define Z_TYPE(zval)            (zval).type  
  13. #define IS_STRING               6  
展开后得到：
[cpp]  view plain copy
  1. const char *__s=('foo');  
  2. (foo).value.str.len=strlen(__s);  
  3. (foo).value.str.val=(duplicate?estrndup(__s, (zval).value.str.len):(char*)__s);  
  4. (foo).type=6;  
ZVAL_STRING做的工作是设置数据类型并赋值。


ZEND_SET_SYMBOL这里只展开一部分宏，因为它展开后过于复杂。
[cpp]  view plain copy
  1. # define EG(v) (executor_globals.v)  
展开部分后得到：
[cpp]  view plain copy
  1. ZEND_SET_SYMBOL(executor_globals.active_symbol_table, 'foo', foo);  
将变量名入当前作用域符号表。


注：之所以使用内核提供的宏来分配内存、注册变量作用域，是因为这样能提高兼容性。
